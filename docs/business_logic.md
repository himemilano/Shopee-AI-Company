# Shopee-AI-Company: 台湾（蝦皮購物）自動化マスター仕様書

## 1. システム概要
本ドキュメントは、M&A（事業売却）を前提とした完全独立型EC自動化システム『Shopee-AI-Company』の根幹となる業務ロジック、価格計算、および例外処理（トラブルシューティング）の仕様を定義する。AI社員（GitHub Actions / エージェント）は、本仕様書に定義されたルールを100%厳守してプログラムを実装・実行すること。

---

## 2. コア一気通貫ワークフロー（通常運用）
AI社員は、以下のステップをAPI連携によって無人で完結させる。

1. **仕入れ元監視 (Amazon API)**
   - 指定されたASIN（商品コード）の価格、在庫ステータスを定期巡回。
2. **クリエイティブ自動生成 (Canva API)**
   - Amazonから抽出した画像を、登録済みの「台湾専用Canvaテンプレート（有料会員枠）」へ自動流し込み。
   - タイトルや「日本直送」等のテキストを自動合成し、正方形（1:1）の出品画像を生成。
3. **現地語SEOリライト (翻訳・生成AI)**
   - Amazonの商品情報を、台湾の購買層に刺さる繁体字（現地表現）に翻訳。
   - 検索ボリュームの大きいハッシュタグ（例：#日本直送 #限定版 など）を末尾に自動付与。
4. **自動出品 (Shopee API)**
   - 生成された画像と文章、および次項のロジックで計算された価格を用いて、台湾ストアへ自動出品（または下書き保存）。

---

## 3. 台湾特化型・自動価格改定ロジック
無在庫販売における利益を絶対防衛するため、以下の計算式に基づき、Amazonの価格変動とShopeeの手数料・税率変更をリアルタイムで同期する。

### 【価格算出公式】
販売価格（NT$） = [ Amazon仕入価格（円） + 国内外送料（円） + 目標粗利（円） ] / [ 為替レート(円/TWD) × (1 - 台湾手数料率 - 決済手数料率 - 参加プログラムサービス料率) ] + 現地調整費（NT$）

- **自動変動トリガー:** - Amazon APIが価格変更を検知した時
  - 為替レートに一定以上の変動（例：1%以上）があった時
  - 各種手数料率（引数管理）が変更された時
- **安全弁（ストッパー）:** 計算された販売価格が、あらかじめ設定した「最低許容粗利」を下回る場合は、出品を自動で「非公開（Delist）」にする。

## 4. 3大トラブル例外処理（ディフェンス・ロジック）

### 例外A：仕入れ元（Amazon）の在庫切れ
- **検知タイミング:** Amazon APIによる定期巡回時（1日4回）。
- **AI社員の自動処理:** - 対象商品が在庫切れ、または入荷待ち（発送まで1週間以上など）になった場合、即座にShopee APIを叩き、対象商品の在庫数を「0」に更新、または「非公開（Delist）」ステータスへ移行させる。
  - Amazon側で在庫が復活したことを検知した場合、自動的に前項の価格計算を行い、在庫を戻して再公開する。

### 例外B：人間による実店舗での商品確保（ローカルエスケープ）
Amazonに在庫がない場合でも、運営者が近隣の実店舗（ヨドバシ、ドンキ等）で商品を物理的に確保した場合の処理。
- **トリガー:** 運営者がGitHubの該当注文Issueに『実店舗確保』のラベルを付与する。
- **AI社員の自動処理:**
  - 仕入れ元の自動追従ステータスを一時的に「手動固定」にロック。
  - 「商品が実店舗から発送、または運営者の元に確保された日（手動入力）」の翌日午前中に、Shopee API経由で自動的に「Paid SPS（集荷依頼）」を実行する。

### 例外C：商品廃盤に伴う取引キャンセル
仕入れが完全に不可能となり、ストア側から注文をキャンセルせざるを得ない場合の処理。アカウントペナルティ（Non-fulfillment Rateの悪化）を回避するため、購入者都合への誘導を自動で行う。
- **トリガー:** 運営者が該当注文に『廃盤キャンセル』フラグを立てる。
- **AI社員の自動処理:**
  - Shopee チャットAPIを起動。
  - 以下の「繁体字テンプレート」をベースに、丁寧かつ同情を誘うお詫びメッセージを顧客へ自動送信。
  - 顧客が「購入者都合」でキャンセル申請を出した場合は、APIがそれを検知して1秒以内に自動承認（Auto-Approve）する。

#### 【チャット送信テンプレート（繁体字）】
「親愛的顧客您好，非常感謝您的光臨。
真的很抱歉，我們剛剛接獲日本廠家的緊急通知，您所購買的商品【商品名】由於極其熱銷，目前已全面斷貨且確認停產（廢盤），導致我們無法順利為您出貨。
為了不耽誤您的寶貴時間，是否能麻煩您從您的訂單頁面申請『由買家發起的取消訂單』呢？收到您的申請後，我們系統將會立即為您辦理全額退款。
對於造成您的不便，我們致以最深切的歉意，懇請您的諒解。謝謝您。」
